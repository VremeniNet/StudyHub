# StudyHub

**StudyHub** — доступный веб-сервис для подготовки студентов к зачётам и экзаменам.

Проект разработан в рамках задания по дисциплине **UI/UX**. Основная цель — придумать продукт с пользовательскими сценариями, описать техническое задание и реализовать интерактивный прототип, который можно открыть и протестировать.

## Описание проекта

StudyHub помогает студенту структурировать подготовку к зачёту: выбрать предмет, увидеть план тем, открыть учебный материал, отметить тему как изученную и отслеживать общий прогресс.

## Идея продукта

Многие студенты готовятся к зачётам хаотично: материалы лежат в разных местах, прогресс сложно оценить, а список тем не всегда очевиден. StudyHub решает эту проблему через простой учебный dashboard.

Пользователь видит:

* список предметов;
* прогресс по каждому предмету;
* рекомендуемую следующую тему;
* краткие материалы;
* чеклист изучения;
* достижения;
* настройки доступности.

## Неформальная постановка задачи

Студенту нужен простой сервис, который помогает понять, что уже изучено, что осталось повторить и какой следующий шаг нужно сделать перед зачётом.

Сервис должен быть понятным, визуально приятным и доступным.

## Формальная постановка задачи

Необходимо разработать SPA-приложение на **React + TypeScript**, позволяющее пользователю:

* просматривать список учебных предметов;
* открывать страницу конкретного предмета;
* видеть список тем для подготовки;
* открывать отдельную страницу темы;
* читать краткий учебный материал;
* просматривать чеклист изучения;
* отмечать тему как изученную;
* видеть прогресс подготовки;
* получать достижения;
* изменять настройки доступности;
* сохранять прогресс между сессиями.

Приложение работает без серверной части. Данные хранятся локально в браузере с использованием `localStorage`.

## Целевая аудитория

Основная аудитория проекта:

* студенты, готовящиеся к зачётам и экзаменам;
* студенты, которым нужен понятный план подготовки.

## Основные пользовательские сценарии

### Сценарий 1. Просмотр общей подготовки

1. Пользователь открывает главную страницу.
2. Видит общий прогресс подготовки.
3. Видит список предметов.
4. Видит рекомендуемую следующую тему.
5. Переходит к продолжению подготовки.

### Сценарий 2. Работа с предметом

1. Пользователь открывает страницу предмета.
2. Видит описание предмета и дату зачёта.
3. Видит список тем.
4. Понимает, какие темы уже изучены, а какие ещё находятся в плане.
5. Открывает нужную тему.

### Сценарий 3. Изучение темы

1. Пользователь открывает страницу темы.
2. Читает краткий материал.
3. Проверяет чеклист изучения.
4. Нажимает кнопку “Отметить как изученную”.
5. Система сохраняет прогресс.
6. Пользователь получает текстовое уведомление об изменении.

### Сценарий 4. Получение достижения

1. Пользователь выполняет учебные действия.
2. Система проверяет условия достижений.
3. Если условие выполнено, достижение становится открытым.
4. Пользователь может посмотреть его на странице достижений.

### Сценарий 5. Настройка доступности

1. Пользователь открывает страницу настроек.
2. Включает высокий контраст.
3. При необходимости включает крупный текст.
4. При необходимости включает уменьшение анимаций.
5. Настройки сохраняются в браузере.

## Реализованные страницы

### Главная страница

Главная страница содержит:

* крупный заголовок проекта;
* общий прогресс подготовки;
* карточку следующего рекомендуемого действия;
* список учебных предметов;
* блок достижений;
* блок с пояснением доступности;
* ссылку на настройки.

### Страница предмета

Страница предмета содержит:

* название предмета;
* описание;
* дату зачёта;
* количество изученных тем;
* прогресс по предмету;
* карточку следующей темы;
* список всех тем подготовки;
* текстовые статусы тем.

### Страница темы

Страница темы содержит:

* название темы;
* статус темы;
* краткий учебный материал;
* чеклист изучения;
* кнопку отметки темы как изученной;
* информацию о примерном времени изучения.

### Страница достижений

Страница достижений содержит:

* общий прогресс по достижениям;
* список открытых достижений;
* список ещё не полученных достижений;
* условия получения достижений.

### Страница настроек доступности

Страница настроек содержит:

* переключатель высокого контраста;
* переключатель крупного текста;
* переключатель уменьшения анимаций;
* описание реализованных accessibility-подходов;
* кнопку сброса прогресса.

### Страница 404

Если пользователь открывает несуществующий маршрут, отображается отдельная страница ошибки с возможностью вернуться на главную.

## Доступность

Проект ориентируется на базовые принципы доступного интерфейса.

В StudyHub реализовано:

* семантическая структура страниц;
* использование `main`, `header`, `nav`, `section`, `article`;
* логичная иерархия заголовков;
* skip-link для быстрого перехода к основному содержимому;
* управление основными действиями с клавиатуры;
* видимое состояние фокуса через `:focus-visible`;
* текстовые статусы вместо передачи смысла только цветом;
* текстовое описание прогресса;
* скрытые пояснения для скринридеров через `.visually-hidden`;
* уведомления через `aria-live`;
* переключатели доступности на Radix UI;
* режим высокого контраста;
* режим крупного текста;
* режим уменьшения анимаций;
* поддержка системной настройки `prefers-reduced-motion`.

## Как достигается доступность

### Клавиатурная навигация

Все основные действия выполняются через стандартные интерактивные элементы:

* `button`;
* `a`;
* `input`-подобные компоненты Radix UI.

Кликабельные `div` не используются для основных действий.

### Скринридеры

Для важных изменений используется `aria-live`. Например, после отметки темы как изученной пользователь получает текстовое сообщение:

> Тема отмечена как изученная.

Прогресс и статусы отображаются текстом, а не только визуально.

### Цвет и контраст

Цвет не является единственным способом передачи смысла. Например, статус темы отображается как текст:

* “Изучено”;
* “В плане”;
* “Статус: тема изучена”;
* “Статус: тема ещё не изучена”.

### Настройки интерфейса

Пользователь может включить:

* высокий контраст;
* крупный текст;
* уменьшение анимаций.

Настройки сохраняются в `localStorage`.

## Хранение данных

Проект использует `localStorage`.

Сохраняются:

* список изученных тем;
* настройки доступности.

Для управления состоянием используется Zustand с middleware `persist`.
