# StudyHub

**StudyHub** — веб-сервис для подготовки студентов к зачётам и экзаменам.

Проект разрабатывается в рамках задания по дисциплине UI/UX. 
Основная цель — спроектировать и реализовать интерактивный сервис с пользовательскими сценариями, техническим заданием, доступным интерфейсом и рабочим прототипом на React + TypeScript.

## Идея проекта

StudyHub помогает студенту структурировать подготовку к зачёту: выбрать предмет, увидеть план тем, отслеживать прогресс, открывать учебные материалы и отмечать изученные темы.


## Неформальная постановка задачи

Студенту часто сложно понять, с чего начать подготовку к зачёту, какие темы уже изучены, какие ещё остались и насколько он готов к сдаче. Необходимо создать простой и понятный сервис, который помогает видеть план подготовки, двигаться по нему шаг за шагом и сохранять мотивацию.

Интерфейс должен быть доступен.

## Формальная постановка задачи

Необходимо разработать SPA-приложение на React + TypeScript, позволяющее пользователю:

- просматривать список учебных предметов;
- открывать страницу предмета;
- видеть список тем для подготовки;
- открывать материалы по теме;
- отмечать тему как изученную;
- видеть общий прогресс подготовки;
- получать достижения за выполненные действия;
- изменять настройки доступности интерфейса.

Приложение должно работать без серверной части. Данные хранятся локально в браузере с использованием `localStorage`.

## Целевая аудитория

Основная аудитория:

- студенты, готовящиеся к зачётам и экзаменам;
- студенты, которым нужен простой план подготовки.

## Основные пользовательские сценарии

### Сценарий 1. Просмотр текущей подготовки

1. Пользователь открывает главную страницу.
2. Видит список предметов.
3. Видит общий прогресс подготовки.
4. Видит рекомендуемую следующую тему.
5. Нажимает кнопку “Продолжить подготовку”.

### Сценарий 2. Изучение темы

1. Пользователь открывает страницу предмета.
2. Выбирает тему из списка.
3. Открывает страницу темы.
4. Читает краткий материал.
5. Отмечает тему как изученную.
6. Получает обновлённый прогресс.

### Сценарий 3. Получение достижения

1. Пользователь выполняет одно или несколько действий.
2. Система проверяет условия достижений.
3. Если условие выполнено, пользователь получает достижение.
4. Интерфейс показывает текстовое уведомление.
5. Достижение появляется в разделе достижений.

### Сценарий 4. Настройка доступности

1. Пользователь открывает настройки.
2. Включает режим высокого контраста.
3. При необходимости включает крупный текст.
4. При необходимости включает уменьшение анимаций.
5. Интерфейс сохраняет выбранные настройки.

## Функциональные требования

- отображение списка предметов;
- отображение прогресса по каждому предмету;
- отображение списка тем внутри предмета;
- возможность открыть страницу отдельной темы;
- возможность отметить тему как изученную;
- сохранение прогресса в `localStorage`;
- отображение достижений;
- настройки доступности;
- навигация между страницами без перезагрузки.

## Нефункциональные требования

- приложение должно быть реализовано на React + TypeScript;
- приложение должно запускаться локально через Vite;
- интерфейс должен быть адаптивным для desktop-экрана;
- интерфейс должен быть понятным и предсказуемым;
- данные должны сохраняться после перезагрузки страницы.

## Требования доступности

Проект должен учитывать базовые требования доступности:

- использование семантических HTML-элементов;
- логичная структура заголовков;
- возможность управления интерфейсом с клавиатуры;
- видимое состояние фокуса;
- текстовое описание прогресса;
- отсутствие передачи смысла только через цвет;
- доступные названия кнопок и ссылок;
- корректная связь `label` и элементов форм;
- уведомления о важных изменениях через `aria-live`;
- режим высокого контраста;
- режим уменьшенных анимаций;
- режим увеличенного текста.

## Как достигается доступность

В проекте используются следующие подходы:

- интерактивные элементы реализуются через `button`, `a`, `input`, а не через кликабельные `div`;
- прогресс отображается не только визуально, но и текстом;
- достижения имеют текстовые названия и описания;
- карточки предметов и тем имеют понятные заголовки;
- после выполнения темы пользователь получает текстовое уведомление;
- цвет не является единственным способом показать статус;
- настройки доступности сохраняются локально;
- компоненты Radix UI используются для доступных интерактивных элементов.

## Основные экраны

### Главная страница

Содержит:

- приветствие;
- карточку рекомендуемого предмета;
- общий прогресс;
- список предметов;
- блок достижений;
- блок текущей серии подготовки.

### Страница предмета

Содержит:

- название предмета;
- описание;
- прогресс подготовки;
- список тем;
- статусы тем.

### Страница темы

Содержит:

- название темы;
- краткий учебный материал;
- список ключевых пунктов;
- кнопку отметки темы как изученной.

### Страница достижений

Содержит:

- список полученных достижений;
- список ещё не открытых достижений;
- описание условий получения.

### Страница настроек

Содержит:

- переключатель высокого контраста;
- переключатель уменьшенных анимаций;
- переключатель крупного текста.
